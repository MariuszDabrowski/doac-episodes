# DOAC episodes

A curated, browsable index of every long-form **Diary of a CEO** episode.
Rewritten titles, editorial summaries, guest credibility lines, topic
tags, lazy-loaded portraits. The catalog is generated by an AI-assisted
ingestion pipeline (Claude Haiku 4.5 + Wikipedia + Open Library + Brave)
and reviewed by hand before each release.

## Stack

| Piece | What | Where |
| --- | --- | --- |
| Frontend | Vite + Vue 3 + Vue Router (static SPA) | `client/` |
| Review server | Hono (local-only, powers `/review`) | `server/index.mjs` |
| Ingest pipeline | Node CLIs + a Python venv for face detection | `scripts/ingest/`, `scripts/portraits/` |
| Data | Committed JSON + gitignored `_*` working files | `data/` |
| Portraits | JPG + WebP + AVIF, served via `<picture>` | `public/portraits/` |
| Specs | Project brief, ingestion process, review page | `specs/` |
| CI | Scheduled ingest + GitHub Pages deploy | `.github/workflows/` |

## Quickstart

```sh
# Prereqs: Node 18+ (`.nvmrc` pins LTS), Python 3.12, ffmpeg, yt-dlp.
nvm use
npm install

# Python venv for face detection + portrait scoring
python3 -m venv .venv
.venv/bin/pip install \
  opencv-python "mediapipe==0.10.18" face_recognition "setuptools<81" \
  Pillow pillow-avif-plugin

# Env vars (only needed for ingestion + `/review`; the SPA itself is static)
# Create a .env at the project root with:
#   ANTHROPIC_API_KEY=...
#   BRAVE_API_KEY=...
#   YOUTUBE_API_KEY=...

# Dev server (Vite on :3000, Hono review API on :3001 via proxy)
npm run dev
```

Open <http://localhost:3000>. The local `/review` route is a triage UI
for editing titles, descriptions, bios, and swapping portrait alts; see
[specs/REVIEW_PAGE.md](specs/REVIEW_PAGE.md).

## Scripts

| Command | What |
| --- | --- |
| `npm run dev` | Vite + Hono review server, concurrently |
| `npm run build` | Build the SPA to `dist/` |
| `npm run preview` | Preview the production build |
| `npm run fetch:channel` | Refresh `data/_youtube-raw.json` from YouTube |
| `npm run ingest <videoId>` | Run one episode through the full pipeline |
| `npm run ingest:batch <count>` | Ingest the next N un-ingested episodes |
| `npm run retag` | Re-run topic-pick on existing episodes (after taxonomy edits) |
| `npm run renumber` | Recompute `episodeNumber` across the catalog |

Internal scripts called by the pipeline or for one-offs:

- `scripts/portraits/auto-portrait.py` — pick the best face crop from candidate frames
- `scripts/portraits/regenerate-portraits.py` — bulk re-run portrait selection
- `scripts/portraits/swap-portrait.py` — promote an alt to primary
- `scripts/portraits/convert-portraits.py` — backfill WebP + AVIF for legacy JPGs

## Automation

A scheduled GitHub Action (every 6 hours) polls the channel for new
long-form uploads, runs the full pipeline on each, and opens a draft PR
with the data + portrait changes. A merge triggers the deploy workflow,
which publishes the new build to GitHub Pages. Setup (secrets, Pages source, PR permissions) is documented at
[.github/workflows/README.md](.github/workflows/README.md).
